# 걸리버 모놀리스 관문 (Gulliver Monolith Gauntlet) — 게임성 설계

> 2026-05-29. 기존 "쇼피스" 빌드(2026-05-27 스펙)가 *기능적으론 완성*이지만 **역경·성취감·중독성이 없다**는 진단에서 출발. 핸드오프의 "유일한 무기 = 작음→거대함의 시각 대비"를 *영상 트릭이 아니라 실제 플레이로 겪는 메커닉*으로 구현한다. 부록 C(공모전 이후 미사일 발전)의 P0(콤보 거대화)를 앞당겨 쓴다.

## 0. 한 줄 요약

작은 비행기로 **솟아오르는 거대 모놀리스(기둥) 숲**을 누비며, *부서지는 기둥(빨간 약점 코어)* 을 연속 명중해 미사일을 **콤보로 거대화**시키고, 마지막 화면을 다 가리는 **거인**을 *충분히 키운 한 방*으로 지운다. 부딪히거나 콤보가 끊기면 약한 채로 도착해 **그 피니시를 못 본다** — 그 아쉬움이 "한 판 더"를 부른다.

## 1. 핵심 판타지 / 심사위원이 상상할 그림

image.png(안개 속, 작은 사람을 짓누르는 거대 콘크리트 monolith)의 스케일 압도감을 *플레이되는 지형*으로. C 서사: **압도적 거인 등장(A) → 작은 미사일론 답답함 → 콤보로 키운 한 방으로 역전(B)**.

## 2. 역경(Adversity) — 이 설계의 심장

기존 빌드의 결함 = 실패할 방법이 없음 = 성취 없음. 역경은 **콤보 체인**에서 나온다.

- 거인은 *충분히 키운 미사일*이라야 끝낼 수 있다. 작으면 칩 데미지만 → **피니시를 못 봄.**
- 미사일은 부서지는 기둥 코어를 *연속 명중*해야 단계적으로 커진다.
- **부딪힘 = HP -1 + 콤보 리셋 + 넉백 → 미사일 도로 작아짐.**
- 즉 거인까지 가는 길 = "체인 안 끊고, 부술 기둥/피할 기둥을 0.5초에 골라내며, 정확히 유지."

핸드오프 안전장치 준수: 죽음보다 *놓친 성취*가 리텐션 레버. fail은 가혹한 즉사가 아니라 HP 3 + 콤보 리셋.

## 3. 기둥(Pillar) 시스템

### 3.1 부서짐 가독성 (절대 원칙)
부서지는 기둥 vs 아닌 기둥은 **한눈에 즉시** 구분돼야 한다(라퓨타 부당사 방지).
- **부서지는 기둥:** 빛나는 **빨간 약점 코어**(emissive RED, 게임 기존 표적 언어와 일치). 코어 명중 → 파괴 + 콤보↑ + (가능하면 길 열림).
- **안 부서지는 기둥:** 통짜 콘크리트(BROWN_DARK), 코어 없음 → 회피 전용.

묘미 = "구분이 어려워서"가 아니라 "구분은 명확한데 골라 쏘고 피하는 손이 어려워서".

### 3.2 동작 4종 (각자 고유 텔레그래프 — 빠를수록 예고 또렷이)
1. **거탑(Colossus):** 느림·압도적 크기. 긴 예고(땅 균열·우르릉). 화면을 채워 *틈을 비집고* 통과. 카메라 뒤로 빠지며 위압.
2. **송곳(Spike):** 빠름·가늘고 높음. 바닥 균열 번쩍 → ~0.3s → 확 솟구침. 반사신경. 솟는 순간 카메라 펀치.
3. **군집 벽(Cluster):** 여러 개 동시 솟아 벽 + 틈 하나. 정밀 레인 통과.
4. **페이크(Fake):** 솟다 멈췄다 플레이어 지날 때 마저 솟음. 2차 예고(깜빡임) 필수. 남발 금지.

### 3.3 난이도 곡선
초반 거탑(스케일 압도 먼저) → 송곳·군집·페이크 혼합(손 바빠짐) → 거인 직전 기둥 폭풍(콤보 유지 최고 압박) → 거인 피니시.

## 4. 콤보 거대화 (부록 C P0 — 앞당김)

- 부서지는 코어 명중마다 `combo++`. 단계 임계(예: 3/6/10)마다 미사일 tier↑.
- **4축 동시 상승(tier에 비례):** ① 크기(시각/광고) ② 파편·폭발 반경(연쇄) ③ 관통(한 발이 여러 기둥) ④ 연출강도(흔들림·슬로모·플래시·연기 규모).
- **리셋:** 충돌 시, 또는 일정 시간 명중 없음(combo timeout, 관대하게). *매 빗맞힘마다 리셋은 안 함* — 수동 조준엔 너무 가혹.
- 기존 distance기반 `weapon_stage` → 콤보 tier 기반으로 전환. HUD에 콤보·tier 표시.

## 5. 거인 피니시 = 스킬 체크 (게이트 아님)

- 거인은 hero zone에 loom-in. 약점(빨간 코어)에 누적 명중.
- **필요 tier 미만이면 칩 데미지만** → 못 죽이고 지나감(피니시 못 봄).
- 충분히 키운 미사일로 약점 누적 → 과충전 → 화면 지우는 피니시(슬로모+대형 연기+플래시).

## 6. 역동적 카메라

- 거탑/거인 등장: 카메라 dolly back/up → 스케일 압도.
- 송곳 솟음: 짧은 펀치(shake).
- 콤보 tier업: 가벼운 줌 펀치 + 플래시.
- 피니시: 슬로모 + 와이드.
- 구현: `_update_camera`에 동적 오프셋(목표 거탑/거인 존재 시 거리 가산)을 lerp로 부드럽게.

## 7. 고급/사실적 폭발 연기 VFX (사용자 추가 요청)

현재 박스 파티클 + 흰 구체 → **레이어드 연기 플룸**으로 업그레이드:
- **불꽃 코어:** 짧고 밝은 emissive(주황/노랑) 스프라이트, 빠르게 페이드.
- **연기 플룸:** 어두운 회색 roiling 연기, 위로 부풀며 느리게 페이드(알파+스케일 업), 약간의 난류(velocity spread).
- **잔해:** 콘크리트 파편(어두운 박스) 튀고 중력 낙하.
- GPUParticles3D 사용(가능 시) — 단일 draw로 다량, 부드러운 알파. 콤보 tier·거인 피니시에 규모(파티클 수/크기/연기 높이) 비례.
- 톤: image.png의 "안개 낀 콘크리트" — 채도 낮은 회색 연기 + 짧은 따뜻한 불꽃 대비 = "고급스러움".

## 8. 비범위 (이번엔 안 함 — 핸드오프 준수)
- 무기 세대 진화(기관총→레이저), 궁극기 게이지, 보상형 광고/IAP/가챠 (메타·수익화 층, 게임성 검증 후).
- 방어 진화 트리(적이 쏘는 주체 전환 = 가장 비쌈, 공격 축 안정 후).
- 사운드(후순위 — 사용자 지시), APK 빌드/광고 컷(게임성 먼저).
- P3 무기 분화/락온 비용화/선도사격.

## 9. 구현 순서
1. Pillar.gd (4종 + 텔레그래프 + 코어 + 충돌). Main 회랑 → 기둥 편대.
2. 콤보 거대화(미사일 scale = tier, 리셋, 4축).
3. 거인 피니시 스킬 체크 + HP fail-state.
4. 역동 카메라.
5. 고급 폭발 연기.
6. 검증(헤드리스+스크린샷) + 커밋.

## 10. 위험 / 튜닝 메모
- 가독성(부서짐 구분)·텔레그래프가 불충분하면 부당사 → 최우선 튜닝.
- 콤보 timeout·HP·필요 tier는 export(GameConfig) 변수로 빼서 실기기 튜닝.
- GPUParticles3D가 headless 더미 렌더러에서 경고/무시될 수 있음 — 시각 검증은 windowed 스크린샷으로.
